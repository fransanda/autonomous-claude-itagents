# Pipeline State (auto-managed by Coordinator)

This file is written by the /itagentsreview Coordinator at the start of each run and updated as it progresses. If the run is interrupted (ctrl-C, crash, power loss), the next /itagentsreview reads this file and resumes from where it stopped.

## Schema

```
mode: normal | full
started: <ISO timestamp>
last_updated: <ISO timestamp>
current_task: <task ID or 'none'>
current_step: building | reviewing | fixing | none
current_agent: <agent name or 'none'>
retry_count: <0-3>
tasks_completed_this_run: <count>
last_compact_at: <task count when last /compact ran>
```

## Lifecycle
- Created at start of /itagentsreview
- Updated before each agent activation
- DELETED at successful end of run
- If exists at start of new run → previous run was interrupted → resume

Do NOT edit this file manually unless the pipeline is genuinely stuck.
